🏗 Architecture Pattern: Schema-Aware Rule Generation

A pattern gaining traction in enterprise data tools: schema-to-rule pipelines. The flow is simple — (1) extract column names + data types from your dataset, (2) pass the schema as structured context to an LLM, (3) LLM returns ranked pl.col() rule suggestions with confidence scores. This maps directly onto data-validation-web's CodeMirror editor. The key insight from Spine Swarm's approach: use a lead agent that understands schema structure to delegate rule generation to specialised sub-agents (one for numeric checks, one for string formats, one for cross-column constraints). This produces better rules than a single-prompt approach.
