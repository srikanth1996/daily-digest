🏗 Architecture Pattern

LLM-as-Linter via a Language Server Protocol (LSP) Bridge — A pattern gaining traction in 2026 is standing up a thin LSP server on the Express backend that proxies diagnostics requests to an LLM. The frontend CodeMirror 6 editor connects to this LSP over WebSocket. When a user types a validation rule, the editor sends the document to the LSP server, which calls the LLM with the current schema context and returns structured diagnostic objects (line, column, severity, message). CodeMirror renders these as inline squiggles. This decouples the AI inference layer from the UI completely, keeps the Vue components clean, and allows the LLM provider to be swapped (OpenAI → Anthropic → local Ollama) without touching the frontend at all.
