🏗️ Architecture Patterns

Pattern: Validation-as-a-Living-System. 2026's emerging pattern treats data validation not as a static rule engine but as a system with AI feedback loops. An agent monitors validation run results over time, identifies which rules fail most frequently, detects patterns in failures (e.g., always fails on weekends = market closure issue), and proposes updated rule logic as a draft for human review. This could elevate data-validation-web from a rule editor into an intelligent data quality platform.

Pattern: Semantic Rule Search. Instead of browsing flat rule lists, users describe what they need in natural language ("rules that check for negative prices in portfolio data") and a vector search over rule descriptions returns semantically similar existing rules. Reduces duplication, helps new team members discover relevant logic faster. Implementation: embed rule descriptions with text-embedding-3-small, store in pgvector/Pinecone, query from the rule search UI.
